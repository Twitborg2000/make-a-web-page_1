Talking like a programmer: Some basic lingo

HTMLstands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain "text content" which describes "what you see" and markup which describes "how it looks". This is what the tags described above do.

HTML - the basic concepts

Tags And Elements: HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags:

HTML markup: (tag)content(tag) The tag, the content and the closing tag together constitutes an element. For example: < {opentag}content > {closetag}.

Why Computers Are Stupid

They interpret instructions literally. Small mistakes, like misspelling or incorrect tags and element construction by a programmer can have a huge impact. often it will mean that the program will not work as intended.

Inline / Block Elements HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Lesson 2: Web Pages, Documents, And Structure

Creating Structured Documents With HTML

Important Things To Take Away:
  1. HTML & CSS are both "Languages"
  2. HTML controls the "structure" of a web page
  3. CSS controls the "style" of a web page (How it looks)

DOM - Document Object Model: A webpage has a tree-like structure. This is what "DOM" refers to. All elements on a web page are boxes. The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. Browsers display this structure as a series of nested boxes (Right click and choose show source or somethin... in Danish that's what is says). The more indented an element is; the more deeply nested it's corresponding "box" will be.

How do you actually do it

Start by coding big boxes first then add the smaller subboxes. The Title and a graphic might be the first box.Then the body containing text and url links with graphics or video or music would be the second big box. Then break the graphics and text and other elements into smaller boxes to code.

Boxes To HTML

Start by using:

<div> </div>

For each natural box which are text, images, video, and so on. Next add place holders so you are reminded what goes in there:
<div> My App </div>

<div> image </div>

<div> text </div>
Next apply style. ex. "class attribute" (like labels on a box) to remind you what is inside. Class names need to make sense:

ex. <div class="title">My App</div>

<div class="app>"

<div class="screenshot">image</div>

<div class="description">text</div>

Lesson 3:Box Sizing

Four Main Points

  1. HTML elements are boxes and each box has 4 components- margins, borders, padding, and content.
  2. Because there are so many components to each box, it can often be hard to get the size of the box right.
  3. There are two techniques you can use to help deal with sizing issues. set sizes interms of percent rather than pixels, or set the box-sizing attribute to border-box for every element.
  4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning Divs are block elements (as opposed to inline elements). By default they take up the entire width of a page. adding the rule [display-flex;] to the appropriate css will over ride this behavior and let divs appear next t eachother.

Code, Test, Refine

A basic rule of coding is: Do not repeat yourself if you do not have to. Avoiding repition has the following advantages:
  • You write less code: This makes your code easier to manage
  • When, not if, you make a mistake, you only have to correct it in one place
  • Fewer lines of code will ususally make the code faster to execute for the computer
Going forward with HTML and CSS the following process will be useful:
  • look for natural boxes
  • look for repeated styles and semantic elements
  • write your HTML
  • apply styles (from biggest to smallest)
  • fix the small things
